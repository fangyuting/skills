否则，审查以下文件：**$ARGUMENTS**

先读取文件内容，然后从以下维度逐一分析：

## 审查维度

### 1. 性能问题
- 不必要的重渲染（缺少 `useMemo`、`useCallback`、`React.memo`）
- 在渲染函数中创建新对象/数组/函数作为 props
- 列表渲染缺少稳定的 `key`
- 大组件未拆分导致局部状态更新触发整体渲染

### 2. TypeScript 类型安全
- 滥用 `any` 类型
- Props 接口定义不完整
- 未处理可能为 `undefined`/`null` 的值

### 3. 代码可读性
- 过长的函数，建议拆分
- 命名不清晰（变量名、函数名）
- 复杂逻辑缺少注释
- 重复代码可以抽取

### 4. React 最佳实践
- `useEffect` 依赖数组是否完整
- 副作用是否需要清理
- 状态设计是否合理（是否需要提升/下沉）
- 表单处理是否规范

### 5. 错误处理
- 异步请求是否有 loading 状态
- 请求失败是否有用户提示
- 边界情况（空数据、网络错误）是否处理

## 输出格式

对每个发现的问题：
1. **问题描述**：具体是什么问题，在哪一行
2. **影响**：为什么这是个问题
3. **修改建议**：给出修改后的代码片段

最后给出一个整体评分（1-10）和优先级最高的 3 个改进点。
